**Goal:** Replace the heuristic join reorder with a cost-based optimizer backed by Iceberg column stats, cardinality propagation, and DP join enumeration.

**Architecture:** Four layers built bottom-up: (1) statistics data structures, (2) Iceberg column stats extraction, (3) cardinality estimation + cost model, (4) DP join reorder. Each layer is independently testable.

**Tech Stack:** Rust, Arrow DataType, Iceberg crate v0.9.0 (DataFile column stats API)

```rust
use std::collections::HashMap;

/// Per-column statistics derived from Iceberg file metadata.
#[derive(Clone, Debug)]
pub struct ColumnStatistic {
    pub min_value: f64,
    pub max_value: f64,
    pub nulls_fraction: f64,
    pub average_row_size: f64,
    pub distinct_values_count: f64,
}

impl ColumnStatistic {
    pub fn unknown() -> Self {
        Self {
            min_value: f64::NEG_INFINITY,
            max_value: f64::INFINITY,
            nulls_fraction: 0.0,
            average_row_size: 8.0,
            distinct_values_count: 1.0,
        }
    }
}

/// Operator-level statistics propagated through the plan tree.
#[derive(Clone, Debug)]
pub struct Statistics {
    pub output_row_count: f64,
    pub column_statistics: HashMap<String, ColumnStatistic>,
}

impl Statistics {
    pub fn avg_row_size(&self) -> f64 {
        if self.column_statistics.is_empty() {
            8.0
        } else {
            self.column_statistics
                .values()
                .map(|c| c.average_row_size)
                .sum()
        }
    }

    pub fn compute_size(&self) -> f64 {
        self.output_row_count * self.avg_row_size()
    }
}

/// Three-dimensional cost estimate (aligned with StarRocks CostEstimate).
#[derive(Clone, Debug, Default)]
pub struct CostEstimate {
    pub cpu_cost: f64,
    pub memory_cost: f64,
    pub network_cost: f64,
}

impl CostEstimate {
    pub fn total_cost(&self) -> f64 {
        self.cpu_cost * 0.5 + self.memory_cost * 2.0 + self.network_cost * 1.5
    }

    pub fn add(&self, other: &CostEstimate) -> CostEstimate {
        CostEstimate {
            cpu_cost: self.cpu_cost + other.cpu_cost,
            memory_cost: self.memory_cost + other.memory_cost,
            network_cost: self.network_cost + other.network_cost,

    }
}
```

- [ ] **Step 1: Add `extract_data_files_with_stats` function to `src/standalone/iceberg.rs`**

Add next to the existing `extract_data_files` function. This function reads manifest entries directly to access DataFile column-level statistics.

The iceberg crate v0.9.0 API path is:
- `table.metadata().current_snapshot()` → `Snapshot`
- `snapshot.load_manifest_list(file_io, metadata)` → `ManifestList`
- `manifest_list.entries()` → `&[ManifestFile]`
- For each ManifestFile, read the manifest to get `ManifestEntry` objects
- `entry.data_file()` → `&DataFile`
- `data_file.column_sizes()`, `data_file.null_value_counts()`, `data_file.lower_bounds()`, `data_file.upper_bounds()`

The function should:
1. Get current snapshot from table metadata
2. Load manifest list
3. For each manifest file, load the manifest and iterate entries
4. For each alive DATA entry, extract path, size, record_count, and per-column stats
5. Map field IDs to column names using the table schema (`table.metadata().current_schema()`)
6. Return `Vec<DataFileWithStats>` where each item has path, size, row_count, and `HashMap<String, IcebergColumnStats>`

Note: `lower_bounds()` and `upper_bounds()` return `&HashMap<i32, Datum>`. The `Datum` needs to be serialized to bytes for storage in `IcebergColumnStats`. Use Datum's binary serialization or convert to `Vec<u8>`.

If the manifest reading fails or no snapshot exists, fall back to the existing `extract_data_files` behavior (path + size + row_count only, no column stats).

```rust
use crate::sql::catalog::{IcebergColumnStats, TableDef, TableStorage};
use arrow::datatypes::DataType;

/// Build TableStatistics from a TableDef by aggregating S3FileInfo column stats.
pub fn build_table_statistics(table: &TableDef) -> TableStatistics {
    let TableStorage::S3ParquetFiles { files, .. } = &table.storage else {
        // Local parquet: estimate from file size
        let row_count = match &table.storage {
            TableStorage::LocalParquetFile { path } => {
                std::fs::metadata(path).map(|m| m.len()).unwrap_or(10000)
            }
            _ => 10000,
        };
        return TableStatistics {
            row_count,
            column_stats: HashMap::new(),
        };
    };

    let total_rows: u64 = files
        .iter()
        .filter_map(|f| f.row_count)
        .map(|r| r.max(0) as u64)
        .sum();
    let total_rows = total_rows.max(1);

    let mut col_stats = HashMap::new();
    for col in &table.columns {
        let col_name = col.name.to_lowercase();
        let mut total_nulls: i64 = 0;
        let mut total_col_size: i64 = 0;
        let mut has_stats = false;

        for file in files {
            if let Some(ref file_col_stats) = file.column_stats {
                if let Some(cs) = file_col_stats.get(&col_name) {
                    has_stats = true;
                    total_nulls += cs.null_count.unwrap_or(0);
                    total_col_size += cs.column_size.unwrap_or(0);
                }
            }
        }

        if !has_stats {
            col_stats.insert(col_name, ColumnStatistic::unknown());
            continue;
        }

        let nulls_fraction = if total_rows > 0 {
            (total_nulls as f64 / total_rows as f64).clamp(0.0, 1.0)
        } else {
            0.0
        };
        let non_null_rows = (total_rows as i64 - total_nulls).max(1) as f64;
        let average_row_size = if total_col_size > 0 {
            total_col_size as f64 / non_null_rows
        } else {
            default_column_size(&col.data_type)
        };
        // NDV heuristic: min(row_count, sqrt(row_count) * 10)
        let ndv = (total_rows as f64).sqrt() * 10.0;
        let ndv = ndv.min(total_rows as f64).max(1.0);

        // TODO: decode lower_bound/upper_bound for min/max (phase 1: use defaults)
        col_stats.insert(
            col_name,
            ColumnStatistic {
                min_value: f64::NEG_INFINITY,
                max_value: f64::INFINITY,
                nulls_fraction,
                average_row_size,
                distinct_values_count: ndv,
            },
        );
    }

    TableStatistics {
        row_count: total_rows,
        column_stats: col_stats,
    }
}

fn default_column_size(dt: &DataType) -> f64 {
    match dt {
        DataType::Boolean => 1.0,
        DataType::Int8 => 1.0,
        DataType::Int16 => 2.0,
        DataType::Int32 | DataType::Float32 | DataType::Date32 => 4.0,
        DataType::Int64 | DataType::Float64 => 8.0,
        DataType::Decimal128(_, _) => 16.0,
        DataType::Utf8 | DataType::LargeUtf8 => 32.0,
        _ => 8.0,
    }
}
```

```rust
fn build_table_stats_from_plan(
    plan: &LogicalPlan,
    catalog: &InMemoryCatalog,
    database: &str,
) -> HashMap<String, TableStatistics> {
    let mut stats = HashMap::new();
    collect_scan_stats(plan, catalog, database, &mut stats);
    stats
}

fn collect_scan_stats(
    plan: &LogicalPlan,
    catalog: &InMemoryCatalog,
    database: &str,
    out: &mut HashMap<String, TableStatistics>,
) {
    match plan {
        LogicalPlan::Scan(s) => {
            let key = s.alias.as_deref().unwrap_or(&s.table.name).to_lowercase();
            if !out.contains_key(&key) {
                out.insert(key, crate::sql::statistics::build_table_statistics(&s.table));
            }
        }
        // Recurse into children
        LogicalPlan::Filter(f) => collect_scan_stats(&f.input, catalog, database, out),
        LogicalPlan::Project(p) => collect_scan_stats(&p.input, catalog, database, out),
        LogicalPlan::Aggregate(a) => collect_scan_stats(&a.input, catalog, database, out),
        LogicalPlan::Join(j) => {
            collect_scan_stats(&j.left, catalog, database, out);
            collect_scan_stats(&j.right, catalog, database, out);
        }
        LogicalPlan::Sort(s) => collect_scan_stats(&s.input, catalog, database, out),
        LogicalPlan::Limit(l) => collect_scan_stats(&l.input, catalog, database, out),
        LogicalPlan::Window(w) => collect_scan_stats(&w.input, catalog, database, out),
        LogicalPlan::SubqueryAlias(s) => collect_scan_stats(&s.input, catalog, database, out),
        LogicalPlan::Union(u) => {
            for input in &u.inputs {
                collect_scan_stats(input, catalog, database, out);
            }
        }
        LogicalPlan::Intersect(i) => {
            for input in &i.inputs {
                collect_scan_stats(input, catalog, database, out);
            }
        }
        LogicalPlan::Except(e) => {
            for input in &e.inputs {
                collect_scan_stats(input, catalog, database, out);
            }
        }
        _ => {}
    }
}
```

Implement the recursive `estimate_statistics` function that walks the LogicalPlan bottom-up and propagates Statistics through each operator. This is the core of the CBO.

The function signature:

```rust
use std::collections::HashMap;
use crate::sql::ir::*;
use crate::sql::plan::*;
use crate::sql::statistics::*;

pub(crate) fn estimate_statistics(
    plan: &LogicalPlan,
    table_stats: &HashMap<String, TableStatistics>,
) -> Statistics
```

Implementation for each node type:

**Scan**: Look up table by name (or alias) in `table_stats`, build initial Statistics.

**Filter**: `output_rows = input_rows * estimate_selectivity(predicate, input_stats)`

**Project**: Pass through with column filtering.

**Aggregate**: `output_rows = min(product(group_key_ndvs), input_rows * UNKNOWN_GROUP_BY_CORRELATION)`

**Join**: Based on join type and key NDVs per the formulas in the spec.

**Sort/Limit/Window/Union/etc.**: Simple propagation rules.

**Selectivity estimation** (`estimate_selectivity`): Walks the predicate expression tree:
- `col = literal`: `1.0 / ndv(col)` (or `PREDICATE_UNKNOWN_FILTER` if no stats)
- `col > literal`: `(max - literal_val) / (max - min)` (or 0.5 if no range)
- `col IS NULL`: `nulls_fraction(col)` (or `IS_NULL_FILTER`)
- `AND(a, b)`: `sel(a) * sel(b)`
- `OR(a, b)`: `sel(a) + sel(b) - sel(a) * sel(b)`
- `IN(list)`: `min(list_len / ndv, 1.0)` (or `IN_PREDICATE_DEFAULT_FILTER`)
- Unknown: `PREDICATE_UNKNOWN_FILTER`

Include comprehensive unit tests for each operator type and selectivity pattern.

```rust
use crate::sql::plan::*;
use crate::sql::statistics::*;

/// Estimate the self-cost of a single operator (not including children).
pub(crate) fn estimate_operator_cost(
    plan: &LogicalPlan,
    own_stats: &Statistics,
    child_stats: &[&Statistics],
) -> CostEstimate {
    match plan {
        LogicalPlan::Scan(_) => CostEstimate {
            cpu_cost: own_stats.compute_size(),
            memory_cost: 0.0,
            network_cost: 0.0,
        },
        LogicalPlan::Filter(_) | LogicalPlan::Project(_) => {
            let input_size = child_stats
                .first()
                .map(|s| s.compute_size())
                .unwrap_or(0.0);
            CostEstimate {
                cpu_cost: input_size,
                memory_cost: 0.0,
                network_cost: 0.0,
            }
        }
        LogicalPlan::Aggregate(_) => {
            let input_size = child_stats
                .first()
                .map(|s| s.compute_size())
                .unwrap_or(0.0);
            CostEstimate {
                cpu_cost: input_size,
                memory_cost: own_stats.compute_size(),
                network_cost: 0.0,
            }
        }
        LogicalPlan::Sort(_) => {
            let input = child_stats.first().copied().unwrap_or(own_stats);
            let rows = input.output_row_count.max(1.0);
            CostEstimate {
                cpu_cost: input.compute_size() * rows.log2().max(1.0),
                memory_cost: own_stats.compute_size(),
                network_cost: 0.0,
            }
        }
        LogicalPlan::Join(j) => {
            let left = child_stats.first().copied().unwrap_or(own_stats);
            let right = child_stats.get(1).copied().unwrap_or(own_stats);
            estimate_join_cost(j, left, right, own_stats)
        }
        _ => CostEstimate::default(),
    }
}

fn estimate_join_cost(
    join: &JoinNode,
    left_stats: &Statistics,
    right_stats: &Statistics,
    output_stats: &Statistics,
) -> CostEstimate {
    use crate::sql::ir::JoinKind;

    match join.join_type {
        JoinKind::Cross => {
            // NestLoopJoin cost
            let left_size = left_stats.compute_size();
            let right_rows = right_stats.output_row_count;
            let right_size = right_stats.compute_size();
            CostEstimate {
                cpu_cost: left_size * right_rows * 2.0,
                memory_cost: right_size * 200.0,
                network_cost: 0.0,
            }
        }
        _ => {
            // HashJoin cost
            let left_size = left_stats.compute_size();
            let right_size = right_stats.compute_size();
            let right_rows = right_stats.output_row_count.max(1.0);

            let build_cost = right_size;
            let probe_penalty = (right_rows / 100_000.0).ln().max(1.0).min(12.0);
            let probe_cost = left_size * probe_penalty;

            CostEstimate {
                cpu_cost: build_cost + probe_cost + output_stats.compute_size(),
                memory_cost: right_size,
                network_cost: 0.0,
            }
        }
    }
}
```

```rust
    }
}
```

The new implementation needs:

1. **`reorder_joins_cbo(plan, table_stats)`** — public entry point. Walks the plan tree. When it finds a chain of INNER JOINs, extracts the join graph and calls the DP solver. Non-INNER joins are left in place but their children are recursively optimized.

2. **Join graph extraction** — `extract_join_graph(plan)` flattens a tree of INNER JOINs into:
   - `relations: Vec<LogicalPlan>` (leaf nodes)
   - `predicates: Vec<(TypedExpr, usize, usize)>` (condition, left_relation_idx, right_relation_idx)
   - Returns None if <2 relations or >10 relations (fallback to heuristic)

3. **DP solver** — `dp_join_reorder(relations, predicates, table_stats)`:
   - Uses `u16` bitmask for relation subsets (supports up to 16 relations)
   - `memo: HashMap<u16, DpEntry>` where `DpEntry = { plan: LogicalPlan, stats: Statistics, cumulative_cost: f64 }`
   - Phase 1: init single-relation entries from base stats
   - Phase 2: enumerate subsets of size 2..=n, try all bipartitions, compute join cost for both directions, keep best
   - Phase 3: return `memo[(1 << n) - 1].plan`

4. **Cost comparison** uses `cardinality::estimate_statistics` + `cost::estimate_operator_cost` + cumulative child costs.

5. **Fallback**: if DP fails (>10 tables, no predicates connecting subsets), use the existing `reorder_joins` heuristic.

Keep the existing `reorder_joins` function renamed to `reorder_joins_heuristic` as the fallback.
